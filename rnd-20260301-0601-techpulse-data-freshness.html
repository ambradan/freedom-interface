TechPulse: data freshness

Il Problema

TechPulse ├и un sistema di predictive workforce intelligence che analizza competenze e trend di mercato. Il problema critico ├и la data freshness: i dati su competenze, tecnologie emergenti e trend del mercato del lavoro diventano obsoleti rapidamente.

тЪая╕П Criticit├а specifiche:
  • Tecnologie emergono e diventano mainstream in mesi, non anni
  • Job posting e skill demand cambiano settimanalmente
  • LLM multi-model hanno knowledge cutoff diversi
  • Supabase Edge Functions necessitano dati real-time per predizioni accurate

Approcci Comuni per Data Freshness

1. Real-time Data Ingestion Pipeline

Strategia: Pipeline ETL continua con multiple sorgenti

  • Job Boards API: LinkedIn, Indeed, Glassdoor (polling ogni 6-12h)
  • Tech News RSS: HackerNews, TechCrunch, GitHub Trending
  • Stack Overflow Trends: API per tag popularity
  • GitHub API: Repository stars, language trends, topic growth
Supabase Edge Function тЖТ Cron trigger тЖТ Ingest тЖТ PostgreSQL + Vector Store

2. Hybrid LLM Strategy con Web Search

Strategia: Combina LLM knowledge con real-time web search

  • Perplexity API: Real-time web search con citations
  • Tavily AI: Search API ottimizzata per AI agents
  • Exa (ex-Metaphor): Semantic search per contenuti tecnici
  • Fallback chain: GPT-4 тЖТ Claude тЖТ Perplexity per dati recenti
React UI тЖТ Edge Function тЖТ LLM routing тЖТ Web search augmentation

3. Incremental Vector Store Updates

Strategia: Embeddings incrementali con time-decay

  • Supabase pgvector: Embeddings con timestamp metadata
  • Time-weighted retrieval: Boost documenti recenti in similarity search
  • Automatic pruning: Rimuovi embeddings >6 mesi per skill volatili
  • Refresh strategy: Re-embed documenti popolari ogni 30 giorni
ALTER TABLE embeddings ADD COLUMN freshness_score FLOAT;

4. Change Detection & Alerting

Strategia: Monitora drift nei dati e allerta

  • Skill velocity tracking: Misura tasso di crescita skill demand
  • Anomaly detection: Identifica spike improvvisi (es. nuovo framework)
  • Confidence scoring: Flag predizioni basate su dati stale
  • User feedback loop: "Questi dati sembrano aggiornati?" тЖТ retrain signal

Implementazione Concreta

1. Edge Function per Data Ingestion (Cron ogni 6h):

// supabase/functions/ingest-fresh-data/index.ts
import { serve } from 'https://deno.land/std@0.168.0/http/server.ts'

serve(async (req) => {
  const sources = [
    fetchGitHubTrending(),
    fetchStackOverflowTrends(),
    fetchJobBoardAPIs(),
    fetchTechNewsRSS()
  ]
  
  const freshData = await Promise.all(sources)
  
  // Generate embeddings con OpenAI
  const embeddings = await generateEmbeddings(freshData)
  
  // Store con timestamp
  await supabase.from('skill_embeddings').insert(
    embeddings.map(e => ({
      ...e,
      ingested_at: new Date(),
      freshness_score: 1.0
    }))
  )
  
  return new Response(JSON.stringify({ updated: embeddings.length }))
})

2. Multi-LLM Router con Web Search Fallback:

// supabase/functions/analyze-skills/index.ts
async function analyzeSkills(query: string) {
  // 1. Cerca in vector store (con time-decay)
  const recentEmbeddings = await supabase.rpc('search_embeddings', {
    query_embedding: await embed(query),
    time_decay_factor: 0.1 // Boost recenti
  })
  
  // 2. Se dati troppo vecchi, usa web search
  const avgAge = calculateAvgAge(recentEmbeddings)
  if (avgAge > 30) { // >30 giorni
    const freshData = await perplexitySearch(query)
    return { data: freshData, source: 'web_search' }
  }
  
  // 3. Altrimenti usa LLM con context
  const response = await callLLM(query, recentEmbeddings)
  return { data: response, source: 'vector_store' }
}
